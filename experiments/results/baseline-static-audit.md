# Baseline static audit: current Nemo Agent landing

Baseline: `shekelstrong/nemo-agent-landing`, копия в `baseline/current/`.

## Что есть в текущем лендинге

- Next.js landing с несколькими страницами.
- Основной фокус: AI-агенты для бизнеса.
- Есть Hero, How It Works, Pricing, FAQ, Footer, Sticky CTA.
- Есть статьи и контентные страницы.
- Есть UI-компоненты: Button, Card, Badge, FilterChip.
- Есть базовые конверсионные элементы: CTA, pricing, FAQ.

## Сильные стороны

- Уже есть полноценная структура лендинга.
- Есть контентная база.
- Есть визуальная система и UI-компоненты.
- Есть понятная тема: автоматизация бизнеса ИИ-агентами.

## Слабые стороны для эксперимента

- Нужно проверить, насколько текущий Hero продаёт измеримый результат, а не просто “ИИ-агенты”.
- Нужно сравнить текущую версию с более сильной UX-архитектурой.
- Нужно проверить, может ли code-first вариант быть легче и быстрее.
- Нужно проверить, может ли финальный synthesis от M3 дать более сильную конверсионную формулировку.

## Критерии сравнения

- Taste.
- Code quality.
- UX clarity.
- Speed.
- Conversion.
- Brand fit.
- Absence of AI-slop.

## Вывод по baseline

Текущий landing — хороший фундамент. Эксперимент нужен не потому, что текущая версия плохая, а потому что multi-model loop может дать более сильную комбинацию: архитектура Nemotron Ultra + код Laguna/Qwen + финальный synthesis MiniMax M3.
